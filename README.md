# VIBE-CADING

> Point. Label. Vibe with AI.

OpenSCAD is powerful but describing geometry to an AI is hard.
"The top face", "that edge near the hole" — language fails where
space is precise.

VIBE-CADING bridges that gap. Load your STL, click a face,
give it a name. The tool outputs a structured JSON that gives
your AI collaborator exact geometric context — normals, coordinates,
semantics — so you can stop describing and start building.

---

The author has an architectural background, and the project code was largely completed with the assistance of an LLM. The author was primarily responsible for project planning and interaction logic design.

AGENT is already capable of creating SCAD files quite well. However, there is a lack of a communication bridge between humans and AI in the SCAD workflow. This is the motivation behind this project.

- [Python 3](https://www.python.org/downloads/)
- [OpenSCAD](https://openscad.org/downloads.html)
- Chrome or Edge (recommended)

## Quick Start

### Windows

1. **Compile and start the server**

   Double-click `build.bat`, or run in Command Prompt:
   ```
   build.bat
   ```
   This compiles `model/example.scad` → `example.stl` and starts the local server.

2. **Open the viewer**

   Visit in your browser:
   ```
   http://localhost:8080/viewer/index.html
   ```

3. **Annotate your model**

   - Click **Monitor STL** — the tool auto-loads the STL and restores any saved annotations
   - Click faces or edges in the 3D view → fill in the label → click **Save JSON**

4. **Start working with AI**

   Open a new AI session and send:
   ```
   Please read model/AI_CONTEXT.md and follow its instructions.
   Current model: example
   ```
   The AI will load the workflow rules, read the current SCAD source and JSON annotations, then wait for your instructions.

---

### WSL / Linux / macOS

1. **Compile and start the server**
   ```bash
   bash build.sh
   ```

2. **Open the viewer**
   ```
   http://localhost:8080/viewer/index.html
   ```

3. Steps 3 and 4 are identical to Windows above.

> **WSL note:** The browser must access the tool via `http://localhost:8080`. Do not open `index.html` directly as a file — the File System Access API does not work with WSL paths.

## Standard Workflow

```
① Edit example.scad (manually or via AI)
        ↓
② build.bat  (Windows)  /  bash build.sh  (WSL/Linux)
   → Compile SCAD → STL, auto-start server
        ↓
③ Browser: http://localhost:8080/viewer/index.html
   → Click "Monitor STL" → loads STL + restores JSON annotations
        ↓
④ Click faces / edges → add labels → Save JSON
        ↓
⑤ Tell your AI: "Add a hole on face_001..."
   (AI reads the latest JSON first, then edits with exact geometric context)
        ↓
⑥ Back to ①
```

## How the AI Integration Works

`model/AI_CONTEXT.md` is a fixed workflow document. At the start of each AI session:

1. Tell the AI: *"Read `model/AI_CONTEXT.md` and follow its instructions."*
2. The AI reads the current SCAD source and JSON annotations from the local server
3. The AI uses annotation IDs (`face_001`, `edge_a`) as geometric references in all code changes
4. After modifying SCAD geometry, the AI automatically updates the JSON — fixing recoverable annotations and flagging others as invalid
5. Invalid annotations appear with a **⚠ warning** in the viewer — delete and re-annotate as needed
